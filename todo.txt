DOCUMENTED GAPS — TESTING & LOGGING (review snapshot)
────────────────────────────────────────────
These are intentional backlog items; Phase 7 shipped CI and minimal tests, but coverage
and diagnostics remain shallow compared to product surface area.

Frontend (Vitest / TS)
- Only ipc/contracts (stable command names) and ipc/magnet (lightweight preflight).
- No React component tests (*.test.tsx); Testing Library is installed but unused.
- No automated check that invoke() payloads use camelCase keys matching Tauri 2 / Rust
  (regressions caught manually or in production).
- No end-to-end tests (Playwright / WebDriver) against a built app.

Rust (cargo test)
- Solid unit tests: validation (magnets), paths (safe_join_under), scheduler windows.
- engine_smoke is a trivial librqbit version assertion; not an integration test of session
  add/remove or IPC.
- Large untested areas: torrent_commands, rss, disk helpers, engine/session wiring,
  queue_control behavior, most Tauri command handlers.

Logging & diagnostics
- Rust: tracing + RUST_LOG with sparse info!/warn!; many failures return String to UI only.
- No structured error correlation (span/trace IDs), no file-backed rotating app log.
- Frontend: in-memory Activity tab only; no persisted log for support/debug sessions.

CI (still strong)
- Lint, typecheck, unit tests, cargo test, Tauri build matrix — keeps builds honest;
  does not substitute for behavioral tests above.

PHASE 8 — TESTING, LOGGING & OBSERVABILITY ROADMAP
────────────────────────────────────────────
Prioritize in order; each tier builds on the last without blocking releases.

Tier 1 — Contract safety net (low effort, high regression value)
[x] Vitest: assert camelCase invoke shapes for torrentAddFile, torrentPause, setTorrentLabel
    (snapshot or explicit keys) so IPC drift fails in CI. (`src/ipc/client.invoke.test.ts`)
[ ] Optional: thin shared JSON schema or codegen from Rust for commands (longer-term).

Tier 2 — Rust unit coverage expansion
[x] Tests for disk.rs / available_bytes_for_path edge cases (mock or temp dirs).
[x] Tests for rss.rs parsing with fixture RSS/XML files (no network).
[x] Tests for watch_folder path listing / dedup logic with temp dirs.
[x] Extend validation tests for torrent bytes edge cases already partially covered.
[x] ipc.rs: SessionSnapshot JSON keys camelCase (serde contract with frontend).
[x] diag_log.rs: append failure creates log line (Tier 4 companion).

Tier 3 — Frontend behavioral tests
[x] Smoke tests for formatInvokeError and any pure helpers extracted from TorrentWorkspace.
    (`src/ipc/invokeError.ts`, `src/ipc/dialogPaths.ts` + `*.test.ts`)
[x] One React test: mock invoke; assert Add flow calls correct command + payload shape.
    (`src/components/TorrentAddInvoke.test.tsx`)

Tier 4 — Logging improvements (when diagnostics matter)
[x] tracing::warn! on selected command Err paths (`map_cmd_err` in `torrent_commands.rs`).
[x] Append critical failures to a rotating log file: `nexttorrent-diag.log` next to settings
    (cap ~512 KiB, rewritten when oversized). (`src-tauri/src/diag_log.rs`)
[ ] Optional: dev-only bridge to expose last N Rust log lines in Activity (advanced).

Tier 5 — Integration / E2E (highest cost)
[x] Config/manifest smoke: `src-tauri/tests/tauri_config.rs` (bundle id substring).
[ ] Optional: Playwright against built UI for critical paths (add magnet mock, etc.).
[ ] Optional: full scripted launch + invoke (needs WebDriver / heavy CI); not added.

NOTES
────────────────────────────────────────────
- “Fully functional modern client” depends on the chosen Rust BitTorrent stack; adjust phases if using FFI (libtorrent) vs pure Rust libraries.
- Legal compliance is the user’s responsibility; this roadmap is technical only.
- Phase 6 queue: **max active uploads** is stored in settings but not enforced by an app-side loop (downloads + stalled timeouts are).
- Phase 7 line “IPC payload validation” meant minimal tests then; Phase 8 Tier 1–4 now adds
  camelCase invoke tests, broader Rust unit tests, UI helper tests, diag log + tracing on
  command failures. Optional items remain in Phase 8 (codegen, Playwright, Activity log bridge).
